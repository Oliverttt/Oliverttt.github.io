How Machine Learning Is Solving the Binary Function Similarity Problem

二进制比对面临的问题

  • 当前论文复现困难
    1. 论文提供的工具在复现时无法达到论文中的结果
    2. 论文中的代码在不同的数据集上就无法运行
    3. 复现一个实验要耗费很长的时间
  • 评估结果不透明
    1. 由于数据集(编译环境、系统配置、正负样本选择)、评估指标、样本处理指标不同，各论文的结果不具有比较性

论文主要贡献

作者选择并比较十种最具有代表性的方法和它们的变种，这些方法包括了计算机安全、编程语言分析、机器学习三个领域。
作者提出一个新的数据集，创建了一个常用的基准。
作者发现简单的方法，例如模糊哈希在简单设定下有好的效果，但是在复杂情境下效果不好。图神经网络在几乎所有任务中都取得了最好的效果，运行时间也是最快的。

The Binary Function Similarity Problem

Definition 定义

  • 二进制函数相似性
    aims at computing a numeric value that captures the “similarity” between a pair of functions in their binary representation, raw bytes (i.e., machine code) constituting the body of the function, as produced by a compiler.
    在其二进制表示、原始表示上捕捉函数级别的“相似性”，其它的论文可能关注更低层次（基本块）或更高层次（整个程序）上的相似性。

  • 什么样的函数是相似的
    For this paper, we consider two functions as “similar” if they have been compiled from the same source code, independently from the compiler, its version, its compilation flags, or even the architecture the function has been compiled to (e.g., x86, ARM).
    作者将从同样源码，不同编译器、不同编译选项、不同编译架构的函数定义为相似的。两个相似的函数可能差别巨大。

衡量函数相似性

  • Direct vs indirect Comparision

    • Direct Comparision
      机器学习方法，需要遍历整个数据集中的函数，并将待分析函数的特征与所有的函数特征进行比对。
      针对上述问题的解决方法是索引策略，使用基于树的数据结构和位置敏感哈希（近似最近邻搜索）等技术预过滤可能相似的候选对象
    • Indirect Comparision 机器学习方法
      将输入特征降维成低维表示，并运用基于距离的方法比较相似性，这种方法可以一次性的将待分析的函数与样本库中的大量特征进行比对，效率更高。
      1. 模糊散列
        低维表示的一个流行例子是模糊散列。模糊散列是由不同于传统密码散列的算法产生的，因为它们被有意设计为将相似的输入值映射到相似的散列。
      2. 嵌入
        语义相似的输入被映射到彼此接近的点。
        机器学习模型的目标是学习如何生成嵌入，使相似函数之间的相似性最大化，并使不同函数之间的相似性最小化。
        1. 代码嵌入
          汇编代码嵌入通常受到它们可以处理的不同指令数量的影响（所谓的词汇表外问题（OOV））。
          指令或基本块嵌入有时被用来通过使用其他算法（如最长公共子序列）来计算函数相似性，或者它们被用作更复杂模型的一部分。
        2. 图嵌入
          这些嵌入可以通过自定义算法或更复杂的机器学习技术生成，如图形神经网络（GNN），可以更好的基于cfg捕捉跨架构的特征。
          图嵌入方法通常还将每个基本块的信息编码到对应的图节点中，以增加更多的表现力。

Function Representations 函数表示

二进制函数本质上是与特定于体系结构的机器代码和数据相对应的字节流。从这些原始输入开始，研究人员使用了多种方法来提取更高级别的信息，这些信息可以用来判断两个函数是否源自同一源代码。该列表按抽象级别的增加排序

原始字节

combine raw bytes with other information obtained from the control-flow graph (CFG) or the call graph (CG)
将原始字节与CFG或者CG的信息结合分析

汇编代码

using disassembled instructions as input, while others compute additional metrics such as “the number of arithmetic assembly instructions in a given function”
使用反汇编指令作为输入，而其他指令则计算额外的指标，如“给定函数中算术汇编指令的数量”

Normalized Assembly规范汇编

Assembly code often encodes constant values (e.g., immediate operands and absolute or relative addresses), which result in a very high number of potential combinations of operations and operands. Assembly normalization is used to abstract away some of this variability, reduce the vocabulary size, and converge all the possible variations of the same operation into a single representation.
汇编代码通常对常量（例如，立即数操作数和绝对或相对地址）进行编码，这会导致大量操作数和操作数的潜在组合。汇编规范化用于抽象掉一些这种可变性，减少词汇表的大小，并将同一操作的所有可能变化聚合到一个表示中。

Intermediate representations中间表示法

work on an even higher abstraction level by lifting the binary representation to an intermediate representation (IR). The use of an IR brings several advantages
通过将二进制表示提升到中间表示（IR），在更高的抽象级别上工作。
(i) 它可以统一语义等价但语法不同的指令的表示
(ii) 它可能会抽象出不同体系结构的不相关工件
(iii) 它允许应用程序分析技术来简化（并聚合）某些代码结构

Structure

许多方法试图捕捉给定函数的内部结构，或函数在整个程序中所起的作用。为了捕捉函数的内部结构，许多方法提取（过程内）控制流图CFG。一些方法使用从基本块获得的数据来丰富CFG，即属性控制流图ACFG，或从函数（例如，寄存器流图）或二进制文件中的上下文（调用图）获得的其他类型的图或信息。最后，一些技术只是从CFG提供的结构中受益，以计算替代特征，例如tracelets（CFG中连续基本块的序列）

数据流分析

在汇编级别实现算术表达式可能会采用不同的形式来实现相同的语义。为了处理这些场景，之前的工作建议首先基于数据流依赖关系计算程序切片，然后将其规范化并用作捕获函数行为的特征。

动态分析

一些方法依赖于动态分析，例如，通过执行成对的函数，并从输入和输出之间的关系中提取特征。其他方法只是从完整或部分执行跟踪中提取语义特征，而其他方法则利用仿真或混合技术

符号执行与分析

与具体的动态执行相反，有些方法依赖符号执行来完全捕获所分析函数的行为，并在所有可能的路径下确定其输入和输出之间的关系

选择的方法

  1. Bytes fuzzy hashing: Catalog1
  2. CFG fuzzy hashing: FunctionSimSearch
    使用SimHash算法计算图的模糊散列
  3. Attributed CFG and GNN: Gemini
    使用GNN（Structure2vec,s2v通过函数ACFG计算函数嵌入），第一个利用GNN和Siamese结构来学习函数相似性
  4. Attributed CFG, GNN, and GMN
    提出了一种新的图匹配模型来计算图对之间的相似性。作者将函数相似性作为实际用例之一进行了探索。这种方法从机器学习社区提出了两个系统安全研究人员尚未研究的前沿模型。此外，本文还显示了有希望的结果。
  5. IR, data flowanalysis and neural network: Zeek
  6. Assembly code embedding: Asm2Vec
    Asm2Vec，NLP模型源自paragraph2VEC的PV-DM变体，是原始word2vec模型的扩展。Asm2Vec引入了更精细的指令级拆分和嵌入结构，以克服汇编指令词汇表外（OOV）问题的局限性。无监督，在单体架构实验中取得了最先进的结果
  7. Assembly code embedding and self-attentive encoder: SAFE
    SAFE使用Lin等人的自注意力sentence encoder编码器学习跨架构功能嵌入。这种方法代表了seq2seq模型中的NLP编码器，与Asm2Vec相比，它是专门为学习跨架构相似性而设计的
  8. Assembly code embedding, CFG and GNN: Massarelli
    使用了与Gemini相同的结构2VEC GNN，但它改变了块级特征，从手动设计的特征切换到无监督的特征。这种方法很有趣，因为它是Gemini的一种改进，它结合了指令级嵌入、基本块编码器和GNN的优点。
  9. CodeCMR/BinaryAI
    该模型将中间表示与NLP编码器相结合以获得基本块嵌入，并将GNN与图嵌入相结合。两个LSTM对函数中的字符串和整数数据进行编码。函数嵌入是三者的串联，二进制模型是端到端训练的。我们注意到，腾讯的这项工作是一位作者之前工作的后续，当与作者联系时，他们解释了新模型如何更准确。
  10. Trex
    基于hierarchical transformer和micro-traces。本文介绍了一个动态组件，它可以提取函数跟踪，这是学习函数语义的基础。作者在这些轨迹上预训练ML模型，并将学习到的知识转移到语义相似的函数上。匹配阶段完全基于静态特征，而生成微轨迹的仿真只需要在预训练期间进行。这个跨架构的解决方案建立在transformer之上，transformer是NLP中最先进的深度学习模型。

机器学习方法实验评估

由于不同深度神经网络的架构参数对实验的影响，不同的超参数设置，对深度学习模型进行实验评估很困难。

  1. 在所有为函数构建嵌入向量的模型中，GNN在所有评估基准和所有任务中都达到了最佳值。
  2. 大多数机器学习模型在AUC上的表现相似，但在排序指标（MRR10和recall@1)上差异很大。
  3. 考虑其他嵌入模型，SAFE比构造无监督特征的GNN具有更好的AUC，在一个特定参数组中的AUC略优于Gemini。
  4. 对于执行直接比较的算法，GMN是所有任务中表现最好的模型。
  5. Zeek的AUC稍低（大型函数除外），而且MRR10和召回率要低很多。

Comparing Trex

  1. 在XO任务上，Trex和Asm2Vec拥有相同的AUC，相似的MRR10和recall@1，这与其它的word2vec变种都一样。但是这些word2vec变种比GNN和GMN的结果要差一点
  2. 这巩固了一个发现，即语言模型在相同的体系结构任务中很强大。然而，与Asm2Vec不同的是，Trex在XA和XA+XO任务中也保持相同的性能，这显示了transformer在（跨语言）跨架构设置中的强大功能。我们确认Trex提高了XO任务（0.94对0.90）和XA+XO任务（0.94对0.91）的安全性能。我们的测试还表明，在GPU上运行Trex的推断比我们的多处理器Asm2Vec实现（100个函数为3.92s vs.8.51s）快，但比在CPU上安全运行（100个函数为3.92s vs.1.46）慢

Comparing different GNNs
Gemini和Li的模型都对GNN的消息传递与聚合层进行的改进，Li对GNN的改进比Gemini使用的GNN(s2v)的效果要好很多。

Comparing different feature set in GNN(s2v)
Gemini使用Structure2vec这一GNN架构，手动选择特征。更复杂、更难提取的特征不一定比更基本的表示更好。200个操作码的ABoW具有Gemini的20倍特征，这导致GNN节点神经网络的输入矩阵更大。

Modelling functions using a textual encoder

SAFE使用基于指令嵌入的句子编码器，AUC优于使用无监督学习的GNN（s2v）。与Gemini相比，AUC相似，但MRR10和recall@1值要低一些。
SAFE在小型函数上效果更好，当最大指令长度从150增加到250时，结果也有所改善。
SAFE面临词汇表外（OOV）的问题，其它的方法例如InnerEye和Mirror采用了不同的反汇编规范化技术来缓解此问题。在SAFE中衡量了OOV指令的影响，x86-64是受OOV问题影响最大的体系结构（少于30%的函数没有OOV问题），可能原因在于其CISC指令集，其次是MIPS和ARM（这两者少于40%的函数含有单独的OOV词汇）。

Asm2Vec and other paragraph2vec models

Asm2Vec与paragraph2vec变体PV-DM和PV-DBOW相比，这三个模型的性能相似，与GNN相比，使用特定的单架构没有任何优势。
实验结果受指令词汇表大小的影响，随机游走数量的影响，并且受到许多实现细节的影响。在训练中，我们选择了1M（1.9M中的）最小间隔为5的指令，大多是数字偏移量或十六进制地址。降低阈值可以改善结果，但也增加词汇表大小与训练时间。在推导时，我们没有改变词汇表，即使这些无监督的方法可以受益于新的词汇表，而不会使结果无效。
三个模型共享相同的词汇表。当改变一个变量固定其他变量时，三个模型都有较高的AUC。但当考虑多个编译变量时，AUC会下降，例如XC任务，其中编译器、其版本和优化会发生变化。

Comparing efficiency
不同模型的训练与测试效率，我们关注测试时间。

迭代时间上，SAFE是机器学习模型中最快的，处理100个函数的时间为1.46秒。
GMN和GNN的运行时间类似，都最慢，但GMN只处理input中的一对函数。具有Gemini特征的GNN（s2v）比操作码特征的版本慢4倍，原因是特征提取时间更长。
Zeek的推导时间也受长的特征提取和处理时间的影响，是所有方法中最慢的。
在RNN变体中，无监督特征的GNN（s2v）速度较慢，这是因为额外的模型复杂性会延长迭代时间，类似的，Trex也由于其NPL模型的复杂性导致其迭代时间长。
Asm2Vec是速度最慢的，需要10次迭代才能提取新的函数嵌入。由于特殊的指令嵌入结构，Asm2Vec比其它paragraph2vec模型都慢。

CodeCMR/BinaryAI evaluation
实验准备：
论文重现了腾讯的CodeCMR。CodeCMR的最终目标是将二进制与源代码匹配，分离出模型中只处理函数二进制的部分。数据集由ARM和x86函数（32位和64位）组成，使用IDAPro预处理。提取的数据包括带有IDA微指令的属性化CFG、ctree整数和函数数据中的字符串。为了基准比较，我们将200个最常见的操作码的字包和80个IDA微码指令的字包作为基本块特征运行了GMN，
实验结果：
使用IDA微码指令BoW的GNN模型比使用操作码BoW的GNN模型具有更高的AUC，第二个模型对大的K值有更高的召回率。BinaryAI/CodeCMR模型的所有指标都高于我们测试的其他模型。如果这些结果得到社区独立研究的验证，这可能是一个非常有前途的研究方向。
创新点：

  1. 在单个模型中组合多种不同的架构（一个NLP编码器、一个GNN和两个LSTM）
  2. 使用端到端策略联合训练所有内容
  3. 作者表示训练策略（例如，使用距离加权抽样）和损失函数（例如，三重损失）发挥了重要作用，可以显著提高性能。

Gemini：Neural Network-based Graph Embedding for Cross-Platform Binary Code Similarity Detection

现有工作面临的问题

  1. 现有算法依赖于近似的图匹配算法，这些算法速度慢、不准确，并难于应用于新的任务

基于每个二进制函数的控制流图CFG构建嵌入，并通过计算嵌入间的距离判断两个函数是否相似。
主要贡献：

  • 第一个由二进制函数生成嵌入向量的基于神经网络的方法
  • 使用Siamese模型训练嵌入模型，预训练模型可以生成嵌入向量用于后续的相似性检测
  • 提出一种再训练方法使预训练模型能够在收到额外的监督下适用于其它的任务
  • Gemini的AUC比其它的基于图匹配的方法都要好

设计相似性检测函数的困难——跨平台二进制检索

  • 由于在实际应用中无法获得二进制函数的源码，因此相似性检测函数应该直接作用于二进制函数。
  • 允许不同架构不同编译条件的二进制，并且捕捉到其相同部分
  • 对于一对相似的函数应该给出高的分数，对于一对不相似地函数应该给出低的分数
  • 计算效率要高
  • 快速适应不同领域的要求

现有技术

成对图匹配

将控制流图CFG中每个基本块的输入输出对（input-output pairs）作为特征，接着计算图的匹配性
discovRE被提出用于更轻量级的符号级别的特征，例如算术指令的数量，call指令的数量，来加速特征提取并提高匹配效率

图嵌入

为了从控制流图学习一种可索引的特征表示，需要将图表示进行编码成为嵌入向量
使用基于局部敏感哈希（LSH）的数据库对特征向量进行索引，以便可以在O(1)时间内执行搜索查询
Genius,图嵌入框架，首先从属性控制流图ACFG中提取原始特征。在ACFG中，每个顶点都是用一组属性标记的基本块，在Genius中使用了6个基本块级别的属性与两个基本块间的属性。然后将每个ACFG转换为一个高维嵌入，并使用位置敏感哈希（LSH）将其存储到哈希表中。因此，为了识别一组与查询函数相似的二元函数，我们只需要找到查询函数的相应嵌入，并在目标语料库中找到附近的嵌入。
重点在于如何将ACFG转化成嵌入向量，Genius采用基于码本的方法嵌入ACFG。它使用一种聚类算法来训练一个码本，该码本由为每个聚类识别的多个具有代表性的ACFG组成。
Genius使用二部图匹配算法测量指定ACFG和码本中每个代表ACFG之间的相似性。因此，这些相似性度量形成了指定ACFG的特征向量

基于神经网络的嵌入生成

更高的准确率
更高的嵌入效率
更快的离线训练

论文模型

现有的图嵌入网络是为了分类任务，但是论文中模型是为了能够区分两个输入ACFG的相似性，设计了Siamese结构，并且在其中嵌入图嵌入网络Structure2vec。
